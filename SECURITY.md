# Security Policy

## Supported Versions

Security fixes are considered for the latest `main` release and the active
development branch. Older releases may not receive fixes.

## Reporting a Vulnerability

Please use GitHub's private vulnerability reporting or security advisory
workflow from the repository **Security** tab when it is available. Do not
open a public issue for an unpatched vulnerability, and do not include secrets
in a report.

Include a concise description, affected path or version, reproduction steps,
impact, and a suggested mitigation when known. Please allow maintainers time
to investigate before public disclosure.

If private reporting is not enabled, contact the repository maintainers through
the private contact method shown on the repository profile. Do not send tokens,
private keys, passwords, or personal data through public issues or pull
requests.

## Scope

This policy covers the source, tools, packaging metadata, workflows, and
documentation shipped by XQ Proof Lab. Local Pikafish binaries, NNUE files,
ChessDB services, and other third-party assets are outside this repository's
control; report their issues upstream as appropriate.
